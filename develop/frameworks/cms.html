WordPress Visual Form Builder

Visual Form Builder представляет собой плагин, предназначенный для того, чтобы сделать процесс создания формы быстрым и легким, так что пользователю нет необходимости редактировать какой-либо PHP, CSS или HTML код. Вы можете добавить поля формы одним щелчком мыши и изменить их порядок при помощи перетаскивания. Он включает в себя анти-спам систему, подтверждения сообщений, которые можно настроить, возможность управлять формой записи в админ-панели WordPress и многое другое. Одной интересной особенностью Visual Form Builder является возможность дублировать формы одним щелчком мыши. Вы также можете вставлять несколько форм в пост или страницу. Плагин имеет довольно обширный раздел FAQ для решения проблем и его настройки.

wordpress.org/plugins/visual-form-builder

Formidable Forms

Formidable Forms еще один плагин, который имеет две версии, Lite и Pro. С помощью его пользователи могут легко создавать формы простым перетаскиванием элементов (drag-and-drop). Он включает в себя расширенные возможности для создания сообщений, страниц и пользовательских типов сообщений с формами, а также возможность управлять и редактировать их с фронтенда вашего сайта. Formidable также интегрируется с reCAPTCHA и Akismet для контроля за спамом.

Breezing Forms

Он может похвастаться удивительным количеством разнообразных функций, включая многостраничные формы, создание полей с различными условиями без знания Javascript, AJAX-загрузка файлов с прогресс баром, и многое другое. Системные требования

Предлагается составить сравнительную таблицу для понимания степени соответствия известных фрэймворков нашим требованиям. Требования в первом приближении таковы:

  1. Авторизация пользователя (автономная+LDAP/AD). Импорт данных пользователя из AD.
  2. Разграничение доступа к проектам.
  3. Корректное отображение клиентской части в браузерах FireFox, Chrome. Возможность работы из остальных не является обязательной.
  4. Установка и эксплуатация системы не должна требовать приобретения какого бы то ни было платного ПО. Использование коммерческого ПО в качестве средства разработки допускается.
  5. Печатные формы должны выводиться в формате PDF, вывод в любых других форматах - опция.
  6. Требования к интерфейсу пользователя.
  7. Должны быть реализованы следующие технологии работы с данными:
    • Иерархические раскрываемые списки
    • Редактируемые таблицы (inline редактирование)
    • Комбо-боксы со множественным выбором (интервал и отдельные записи)
    • Сортировка табличных форм по любому полю
    • Скрытие/отображение полей в табличных формах
    • Контекстные меню действий над выборкой объектов
    • Групповое редактирование выборки объектов
    • Ускоренный спуск по спискам при начале набора
    • Фильтр по фрагменту строки в пределах поля
  8. Аудит. По каждому изменению объектов системы должен вестись аудит – кто, когда и что изменил.
